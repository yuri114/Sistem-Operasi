KONDISI SAAT INI (setelah Tahap C)
------------------------------------------------------------
- Arsitektur  : x86_64 64-bit Long Mode (IA-32e)
- Resolusi    : 1920×1080 @ 32bpp (VBE linear framebuffer ~8.3MB)
- Memori      : 4-level paging, identity-mapped 4GB, heap 2MB (0x100000–0x2FFFFF)
- Multitasking: round-robin, MAX_TASKS=16, context-switch 64-bit
- Filesystem  : MFS2 — 32 file, 16KB/file
- GUI         : Window Manager (16 window), taskbar, 5 built-in apps
- User space  : Ring-3 isolasi penuh, ELF64 loader, user stack @0x600000
- Kernel size : ~178KB
- Platform    : QEMU qemu-system-x86_64, build via WSL cross-compiler (x86_64-linux-gnu)

TAHAP C — 64-BIT LONG MODE (penulisan ulang arsitektur)
Status: [SELESAI] ✓ Selesai 26 Maret 2026 — build 178 KB, boot ke Long Mode, GUI stabil
Prasyarat: Tahap B/B+ selesai ✓
----------------------------------------------------------------
Tujuan     : Migrasi dari i386 32-bit Protected Mode ke x86_64
             IA-32e Long Mode. Penulisan ulang penuh semua
             komponen low-level kernel.

Sub-tahap:

  [C1] Bootloader & kernel entry 64-bit                           ✓
       - boot.asm: Real Mode → Protected Mode (tidak berubah)
       - kernel_entry.asm [BITS 32]: zero BSS, setup 4-level page
         tables (PML4/PDPT/4xPD) di 0x1000–0x6FFF identity-map 4GB,
         aktifkan PAE + EFER.LME + CR0.PG, load GDT64, far jmp
       - kernel_entry.asm [BITS 64]: reload segs, RSP=0x90000,
         call kernel_main()
       - GDT64: null / kernel CS 0x08 / kernel DS 0x10 /
                user CS 0x1B / user DS 0x23 / TSS64 0x28 (16-byte)
       - Compiler: nasm -f elf64, x86_64-linux-gnu-ld -m elf_x86_64

  [C2] Linker & build system 64-bit                               ✓
       - linker.ld: OUTPUT_FORMAT(elf64-x86-64), . = 0x8000
       - build.ps1: x86_64-linux-gnu-gcc -m64 -mno-red-zone
         -mcmodel=small -fno-pic -no-pie, nasm -f elf64
       - QEMU: ganti qemu-system-i386 → qemu-system-x86_64

  [C3] IDT & ISR 64-bit                                           ✓
       - idt.h / idt.c: gate descriptor 16-byte, offset_hi 32-bit
       - isr.asm [BITS 64]: SAVE_REGS/RESTORE_REGS 15 GPR
         (rax,rcx,rdx,rbx,rbp,rsi,rdi,r8-r15)
       - irq0 timer: context switch current_rsp/next_rsp (uint64_t*)
       - irq1 keyboard, irq12 mouse
       - int80_handler: [rsp+112]=rax→rdi syscall dispatch
       - exc_common: EXC_NOERR/EXC_ERR macros → exception_handler

  [C4] TSS 64-bit                                                  ✓
       - tss.h: struct TSS64 (rsp0/rsp1/rsp2, IST[7], iomap_base)
       - tss.c: isi 16-byte GDT descriptor, ltr 0x28
       - tss64_set_kernel_stack() dipanggil di setiap task_switch

  [C5] Multitasking 64-bit                                         ✓
       - task.h: uint64_t rsp, uint64_t *page_dir
       - task.c: current_rsp/next_rsp (uint64_t*), iretq frame
         ring-0 (3 qword: RFLAGS/CS/RIP) dan
         ring-3 (5 qword: SS/RSP_user/RFLAGS/CS/RIP)
       - task_switch: tss64_set_kernel_stack + vmm_switch_dir

  [C6] VMM 4-level paging                                          ✓
       - vmm.c: PMM bitmap 4096 frame (16MB), frame 0-767 pre-marked
       - vmm_create_page_dir: PML4→PDPT→pd_low, boot PDs shared
         pd_low[0]=0x000000|0x87 (0–2MB), pd_low[1]=0x200000|0x87 (2–4MB)
       - vmm_map_page: walk 4-level, skip 2MB large pages (bit PS)
       - vmm_free_user_memory: free PTs + frame ≥ 3MB + pd_low + pdpt + pml4
       - paging.c: paging_init = no-op (4-level aktif dari kernel_entry)

  [C7] ELF64 loader                                                ✓
       - elf_loader.h/c: Elf64_Ehdr/Elf64_Phdr, e_machine=0x3E
       - Load address: 0x400000 (pisah dari kernel heap)
       - User stack: 0x600000 (1 frame 4KB)

  [C8] Kernel C & syscall 64-bit                                   ✓
       - kernel.c: enter_usermode iretq, exception_handler uint64_t
         tss64_init, print_hex64
       - syscall.c: is_user_ptr(uint64_t), semua return uint64_t,
         kernel_exec / SYS_EXEC memakai uint64_t proc_dir/entry/stack
       - lib.h: syscall0/1/2 return long, (long) pointer cast
       - user.ld: OUTPUT_FORMAT(elf64-x86-64), . = 0x400000
       - memory.c: saved_flags uint64_t (pushf/popf 64-bit)
       - shell.c: uint64_t proc_dir/entry/sp1/sp2
       - paging.c: paging_get_cr0 uint64_t temp register

TAHAP C — 64-BIT LONG MODE (penulisan ulang arsitektur)
Status: [DIKERJAKAN] ← sekarang (26 Maret 2026)
Prasyarat: Tahap B/B+ selesai dan stabil
----------------------------------------------------------------
Tujuan     : Migrasi dari i386 32-bit PM ke x86_64 Long Mode.
             Ini adalah penulisan ulang besar, bukan modifikasi.

Sub-tahap:

  [C1] Bootloader baru (16-bit → Long Mode)
       - real mode → protected mode → long mode
       - Setup PAE: CR4.PAE=1
       - Setup EFER: IA32_EFER.LME=1
       - 4-level page tables: PML4 → PDPT → PD → PT
       - Load kernel 64-bit, jump ke entry point 64-bit
       - File: src/boot/boot.asm (rewrite total)

  [C2] Kernel entry 64-bit
       - kernel_entry.asm: [BITS 64], register 64-bit
       - GDT 64-bit (base/limit diabaikan, hanya attr yang berlaku)
       - File: src/kernel/kernel_entry.asm (rewrite)

  [C3] IDT 64-bit
       - Gate descriptor 16 byte (bukan 8 byte)
       - ISR wrapper: push rax/rbx/rcx/rdx/rsi/rdi/r8-r15
       - File: src/kernel/idt.c, src/kernel/isr.asm (rewrite)

  [C4] Paging 64-bit
       - 4-level paging: PML4[512], PDPT[512], PD[512], PT[512]
       - Virtual address space: user di 0x0000_0000_0000_0000
                                kernel di 0xFFFF_8000_0000_0000
       - Canonical address 48-bit
       - File: src/kernel/paging.c (rewrite)

  [C5] Multitasking 64-bit
       - Context switch: save/restore 64-bit registers
       - TSS 64-bit (IST entries untuk exception stack)
       - File: src/kernel/task.c, task.h, tss.c (rewrite)

  [C6] Syscall via SYSCALL/SYSRET
       - Menggunakan instruksi SYSCALL (lebih cepat dari int 0x80)
       - LSTAR = alamat kernel syscall handler
       - File: src/kernel/syscall.c, lib.h (update calling convention)

  [C7] Compiler target
       - Saat ini: -m32 (gcc 32-bit)
       - Nanti: -m64 atau target x86_64-elf (perlu cross-compiler baru)
       - build.ps1: ganti semua -m32 → -m64 (-mcmodel=kernel dll.)

  [C8] Memory map 64-bit dengan UEFI (opsional)
       - Saat ini boot lewat BIOS (INT 13h)
       - Untuk hardware modern: bisa pakai UEFI GOP untuk framebuffer
         (tidak butuh Bochs VBE lagi)
       - Ini opsional — BIOS/Bochs VBE tetap bisa dipakai di QEMU
         bahkan untuk 64-bit

Estimasi waktu: 2-4 minggu tergantung kompleksitas
